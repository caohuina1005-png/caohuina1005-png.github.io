The QFM project was not a dashboard upgrade — it was the reconstruction of Chewy's entire quality performance governance foundation. The legacy QPD had lost organizational trust, caused fulfillment centers to build shadow tracking tools, and left leadership without a reliable network-wide view across 40 KPIs.

The Problem — Three layers of failure

  • Data integrity breakdown: KPI logic had drifted from business reality over time; stakeholders couldn't trust the numbers
  • Network alignment collapse: individual FCs built their own local tools with inconsistent metric definitions, making cross-site comparison impossible
  • Decision speed: the flat, undifferentiated layout forced managers to manually scan hundreds of cells to find issues

What I built

  • Audited and revalidated all 40 KPIs against current business logic through structured stakeholder conversations across 5 operational domains (Building, Inbound, Outbound, Inventory Control, Replan)
  • Rebuilt the 5,000-line monolithic SQL script into a modular dbt architecture on Snowflake — each metric isolated in its own model with incremental processing and full lineage visibility
  • Designed a three-tier performance classification system (Healthy / Watch / Critical) encoding judgment directly into the reporting layer
  • Built a role-stratified Tableau workbook: executive heatmap for rapid network scan, operational drill-down for in-week issue identification, and analyst-level single-metric deep dive

Outcome

  • Stakeholder feedback: "a thousand times better" than the legacy QPD with documented instances of the dashboard directly driving FC-level operational decisions
  • Restored a single trusted source of truth across the fulfillment network, eliminating shadow tracking fragmentation
  • Random Defect Rate — a SOX-compliance metric — now produced by a governed, auditable pipeline
Chewy · Predictive Analytics · Data Science
Predictive Risk Scoring Framework — Inventory Audit Optimization
4.5MCycle counts optimized
70%Labor reduction target
0.73AUC-ROC

Inventory accuracy is foundational to fulfillment operations — yet the organization was conducting 4.5 million audits annually through entirely random sampling. The process was labor-intensive, expensive, and untargeted. The question was: could we make auditing risk-driven instead of random?

The insight

  • While the audit process was random, the operational behaviors leading to inventory errors were not — patterns in transaction activity could predict which locations were most likely to have defects
  • Designed a "workflow window" model: for each audit, captured all operational activity between consecutive audits at the same location, creating structured observations with ~20 engineered features

Statistical validation

  • Ran exploratory analysis and discovered defect patterns varied dramatically across facility types — requiring separate models per region
  • Identified time-between-audits as a confounder and controlled for it in the model
  • Applied Chi-square tests for binary signals and Welch's t-tests for continuous features to filter for statistically significant predictors
  • Ran Pearson correlation analysis across all features to check for multicollinearity

Modeling & evaluation

  • Trained a logistic regression model using validated features with standardized scaling, producing a composite risk score per location
  • Achieved 0.73 AUC-ROC on the test set; top 10% highest-risk locations showed a 2x lift in defect rate — same labor, twice the detection

Outcome

  • Transformed the audit workflow from random sampling into targeted, risk-based execution
  • Defect detection efficiency improved within two months of launch
  • Targeting a 70% reduction in audit-related labor, projected to generate significant annual cost savings

MNR Control Tower — Middle-Mile Visibility System
~55%MNR DPMO reduction
5-dayMaturity window logic
Network→Operator level

Shipments were being marked "shipped" in the warehouse system but in some cases never entered the carrier network. From an operations view: no visibility at the warehouse-to-carrier handoff, no standardized metric, and no way to quantify the scale of the problem.

Problem definition & metric design

  • Defined two MNR rules with stakeholders: shipments with only "Out for Collection" events, and shipments with 120+ hour tracking gaps
  • Identified a false-positive issue — newly shipped packages incorrectly flagged due to tracking delays — resolved with a 5-day maturity window

What I built

  • Designed a full SQL pipeline integrating shipment data, carrier tracking, process path info, packaging profiles, and manual handling flags
  • Built a hierarchical dashboard from network level down to FC, process path, profile type, manual vs. automated, and individual operator
  • Designed as a diagnostic tool — not just reporting — so stakeholders could identify where issues were and why

Outcome

  • Surfaced that manual handling flows consistently had higher MNR rates than automated sorter flows, enabling targeted interventions
  • Strengthened carrier performance management with data-backed insights
  • Shifted the organization from reactive issue handling to proactive monitoring
